Drop Python 2.4

What is this about?

Python 2.4 reached the end of its life on 19 December 2008. It's about time to drop support for Python 2.4.

Reasons for dropping

  • pip no longer supports Python 2.4
  • Coverage no longer supports Python 2.4
  • Requests no longer supports Python 2.4
  • Virtually no PyPI traffic (June 2016)

What is this list?

This site shows the top 360 most-downloaded packages on PyPI showing which have dropped support for Python 2.4.

  • Green packages have dropped Python 2.4,
  • White packages may still support Python 2.4.

Packages that are backports (for example, enum34) or known to be deprecated are not included (for example, distribute). If your package is incorrectly listed, please create a ticket.

This is not an official website, just a nice visual way to measure progress. To see the authoritative guide on wheels and other aspects of python packaging, see the Python Packaging User Guide.

My package is white. What can I do?

Remove the classifier

Remove the Trove classifier from setup.py.

'Programming Language :: Python :: 2.4'

Stop testing 2.4

Remove Python 2.4 from your CI. For example Travis CI's .travis.yml:

python:
 - 2.4

And for example from Appveyor's appveyor.yml: 

C:\Python24
C:\Python24-x64

And tox.ini: 

envlist=py24

Remove old code and documentation

Remove old Python 2.4-specific code and documentation. Common files to check:

  • .travis.yml
  • appveyor.yml
  • README.md
  • setup.py
  • tox.ini

Search your code for stuff like: 

if sys.version_info < (2, 5):
    # Python 2.4 stuff

if platform.python_version == "2.4":
    # Python 2.4 stuff

ver = platform.python_version_tuple()
    if float('{0}.{1}'.format(*ver[:2])) < 2.5:
    # Python 2.4 stuff

try:
    # Python 2.4
    import something
except ImportError:
    # Python 2.5+
    import something_else

// In C code
#if PY_VERSION_HEX < 0x02050000
-#endif

Also search for 2.4 and 24.

If you test with coverage, look for code which was tested before removing 2.4 from your CI.

Finally, consider dropping support for Python 2.6 and 3.3, which reached EOL on 2013-10-29 and 2017-09-29 respectively.
